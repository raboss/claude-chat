# services/ — الخدمات

طبقة البنية التحتية: الاتصال بالسيرفر، المصادقة، الترجمة، التخزين.
**كل module في النظام يعتمد عليها — لكنها لا تعتمد على أي module.**

## الملفات

| الملف | الوظيفة | الحالة |
|---|---|---|
| `config.js` | إعداد السيرفر + localStorage proxy | ⚠️ proxy معقّد — سيُنظَّف في Phase 3 |
| `storage.js` | تحميل / حفظ / مزامنة البيانات + merge ذكي للمشاريع | ✅ PRESERVE كامل |
| `auth.js` | المصادقة وتسجيل الدخول | ✅ PRESERVE كامل |
| `i18n.js` | الترجمة AR↔EN (قاموس 1,400+ إدخال) | ✅ PRESERVE — سيُكمَّل لاحقاً بما ينقص |

## قواعد

- ترتيب التحميل في HTML: `config → i18n → storage → auth` ثم الـ modules.
- أي module يقرأ من السيرفر **يجب** أن يمرّ عبر `storage.js`.
- لا يُستخدم `fetch` مباشرة في أي مكان آخر.
